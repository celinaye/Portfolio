Overview

The goal of the redesigned webpage is for the users to view the homepage of Hilo Farmers Market without the problems the current Hilo Farmers Market website faces today. The user can also view the webpage on a phone, tablet, and desktop with ease, as the webpage is responsive to all screen sizes.



Problems Detected

The current Hilo Farmers Market website includes usability, learnability, memorability, accessibility and responsiveness issues that are listed below.

Usability
  • Contains information in the long home page that could be contained in separate tabs
  • Has two different menu bars, with tabs that have similar labels
  • Includes a search bar with no functionality or error prevention
  • Has inconsistent fonts throughout the website
  • Uses a displeasing aesthetic with a extensively filled page
Learnability
  • Has an abundant amount of tabs which makes it difficult for users to navigate on the website
  • The inconsistent font color and sizes does not help the user create patterns or a mental key/legend
  • Includes a long home page that does not help users directly find what they are looking for
  • Does not include any help documentation
Memorability
  • Contains a large amount of tabs and information on the page which creates difficulty for users to relearn how to use the website
  • Links to visuals instead of showing, such as the map, and does not minimize the user's memory load
  • Inconsistency of font and font color forces user to relearn the meaning of the differences
Accessibility
  • 4 images missing alt texts
  • 1 empty label and 1 missing label
  • 1 empty heading and 1 empty link

Lo-Fi Wireframes

First, I created lo-fi wireframes of my redesigned website, noting the specific changes I made and why, and addressing the problems in the current website. Wireframes were created for each of the screen sizes (mobile, tablet, and desktop).

Visual Design Style Guide

Next, I created a visual design style guide to help develop the lo-fi wireframes into hi-fi prototypes in Figma. This included the logo, icons, and modal buttons that will be used in the homepage.

Hi-Fi Prototypes

Finally, high fidelity prototypes were created with Figma to detail the specific main elements of the website. This included noting the changes on each screen size to address the responsiveness of the website. Specifically, by indicating the areas of the page in which a media query should be used.



Conclusion

Throughout the project, I learned the importance of usability, learnability, and memorability principles when designing a website. I learned how to analyze and identify flaws in an existing interface, create low-fidelity and high-fidelity prototypes for various screen sizes, and build a responsive website, using HTML and CSS, based on those prototypes. The project allowed me to practice the workflow of redesigning a simple website.
